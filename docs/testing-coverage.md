# Testing Coverage Matrix

Messages for AI has a few different safety nets because the risky failures do
not all live at the same layer. Fast deterministic checks should block PRs.
Installed-app, visual, and large-data checks can run nightly or manually until
they are fast and stable enough to gate every PR.

## PR-Gated Lanes

| Surface | Primary Check | Coverage Contract |
| --- | --- | --- |
| Menu bar app | `swift test` | Safety gates, scheduling, settings migration, analytics allowlist, preview/export seams, parser seams. |
| Bun MCPs/generators | `bun --bun tsc --noEmit` + `bun test` | Storage, daemon RPC, send floors, read bounds, untrusted wrapping, generator logic. |
| Skills | `python3 -m unittest discover -s tests` | Local skill logic, privacy guards, aggregate-only outputs, voice/birthday/texting analytics behavior. |
| Experience evals | `scripts/test-experience.sh` | Cross-boundary promises: MCP stdio handshake, Wrapped highlight invariants, model-output parser tolerance, site/update metadata. |

## Synthetic Fixtures

Shared public-safe texting fixtures live under `tests/fixtures/texting/`. They
must stay synthetic: no real message bodies, handles, contact names, phone
numbers, emails, or chat exports. Texting Wrapped currently consumes
`wrapped-personas.json`, generated by `generate-wrapped-fixtures.mjs`. Wrapped
evals assert archetype spread, personalized evidence, stricter texting-age
guardrails, and that slang/reply speed do not drive age. Future features should
add fixtures here when they need reusable persona, privacy, or scale coverage.

## Nightly / Manual Lanes

| Surface | Check | Why It Is Not PR-Gated Yet |
| --- | --- | --- |
| Installed app | Dev-bundle smoke and launch smoke | Requires macOS app execution and may depend on local permissions. |
| Visual Wrapped cards | Screenshot/accessibility smoke | Useful but more brittle across rendering engines and font changes. |
| Large-data scale | Synthetic chat history budgets | Valuable before scale, but should run out-of-band until runtime is stable. |
| Release/notarization | `release.sh --dry-run` plus James-run release | Notary/App Management require James's Terminal/keychain context. |

Run `scripts/smoke-installed-app.sh` after a dev install to validate the installed
bundle's sidecars, shared backend/launcher split, Sparkle symlink preservation,
and stdio MCP initialize responses.

## Regression Rule

Every P0/P1 bug fix should include one of:

- A deterministic unit test.
- A deterministic experience eval.
- A nightly/manual smoke entry with the reason it cannot safely run in PR CI.

If none is practical, the PR must explicitly say why and name the residual risk.
